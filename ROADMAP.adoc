= Game Server Admin (GSA) Roadmap
:author: Jonathan D.A. Jewell
:revdate: 2026-03-29

== Current State

GSA is a universal game server probe, config management, and administration tool.
93% complete. Gossamer GUI + Zig FFI + Idris2 ABI + VeriSimDB backing store.

Key capabilities delivered:

* 22 exported FFI symbols (Zig 0.15.2, libgsa.so)
* 8 FFI modules: lifecycle, probe, config_extract, a2ml_emit, verisimdb_client, server_actions, game_profiles, groove_client
* Protocol fingerprinting (8 protocols, 20+ known ports)
* Config parsing (8 formats: XML, INI, JSON, ENV, YAML, TOML, Lua, KV)
* 17 game profiles (A2ML files in profiles/)
* 7 Gossamer GUI panels + 9 panel clades
* VeriSimDB on 2 ports (main 8090, backup 8091 for game saves)
* A2ML serialisation, parsing, and config diff
* Groove voice alerting via .well-known
* Server actions via Podman/Docker/systemd
* Secret redaction in A2ML output and octad JSON
* Chainguard Wolfi containers with quadlets
* Idris2 ABI with 13 result codes
* Desktop entry and icon assets (SVG + PNG)

Test suites:
* Zig unit tests: passing
* Integration tests: 36 tests, passing
* E2E smoke tests: 8/8 against live VeriSimDB
* Gossamer chain: 23/25 (2 Ephapax parser gaps)
